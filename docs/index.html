What's this

This is a collection of column-staggered and ortholinear ergonomic keyboards.

It is not as large as some other collections, but you can sort and filter the keyboards.

"The rules"
  • Column-staggered and ortholinear keyboards only.
    This avoids some ambiguity with counting columns and similar.
  • Keyboards have to be buildable or obtainable in some way - no teasers.
Where else to look for keyboards
  • golem.hu
  • kbd.news
  • MechDB
  • Keebfolio

Frequently asked questions

A keyboard is missing!

Please fill out and submit keyboards using the above editor - I try to add interesting keyboards when I encounter them, but there are hundreds of open-source keyboards alone so help is always welcome.

Generally it takes a few minutes (or even less if you made the thing and know the specifics) to fill out the metadata for a keyboard.

Something's wrong / amiss about a keyboard

You can submit corrections as simple notes or by loading an existing keyboard in the above editor, making edits, and submitting the updated JSON.

Personally I tend to go over 5-10 keyboards at a time based on readily available information, so if something cannot be derived from the description, photos, or part list, I won't be filling that out.

And if you make open-source hardware and are reading this, if you do not have a part list/feature list in your repo, please at least mention whether the keyboard is made for hotswap sockets or not, and the switch type(s) if it's not just MX. The reader shouldn't have to look at KiCAD files .

Can you add a column?

I can generally add additional columns to the table so long as it's something that:

  • Can be easily derived
  • Is of value to the average reader
  • Appears on more than just a couple keyboards

Bonus points if you're willing to go over the existing keyboards and add a new bit of metadata to them yourself.

Some examples:

  • Microcontroller type could be valuable, but is not commonly mentioned in open-source projects - you might recognize from a photo or guess based on slot size, but otherwise you'd have to look at the source files.
  • Splay could have been measured in degrees, but this adds a bunch of columns and a certain amount of overhead (in measuring angles between columns) when going over keyboards with splay.
  • Measuring tilt and tenting is problematic, but kind of important for unibody keyboards, so that gets a pass.
